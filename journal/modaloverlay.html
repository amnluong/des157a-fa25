Modal/ Overlay/ Dialog Window

Website Form Usability Thoughts

10.22.25

A modal window is an element that sits on top of an application’s main window. It disables the main window but keeps it visible with the model window as a child window in front of it. You may want to use the modal window when you need to grab the user’s attention, need user input, show additional information in context, and show additional information that is not in context. When you use the modal window, you need to include an escape hatch, which can be either a cancel button. escape key, or be able to click outside the window to leave it. There should also be a descriptive title so users can know where they are because they are still on the original page. Buttons should have text to know what that button does. Keep the size of the modal in mind, don’t make it too big or small and have the background use the lightbox effect so the modal is the main focus. Only have the modal pop up when the user does an action to make the modal pop up. Modals should not be used on mobile devices since it does not fit well on the screen.
